Asymmetric Cryptography encompasses two keys: a public key and private key.
# [[Hard Problems]]
Asymmetric cryptography uses hard problems to enable encryption and ensure its security.
# [[Asymmetric Ciphers]]
Asymmetric Ciphers are used to encrypt messages with the public key of a person and decrypt it with the respective private key.
# [[Key exchange]]
With asymmetric crypto, we can now exchange keys between to persons where the public key is verified by a signature for example. The shared key can then be used for faster, more efficient [[symmetric cryptography]].
# [[Elliptic Curves]]
Elliptic curves serve as a mathematical basis for asymmetric encryption based on the discrete logarithm problem.
# [[Signatures]]
Signatures enable authentic public keys with public key infrastructure. This way we can trust a public key of a server for example.
